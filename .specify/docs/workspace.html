Input Methods

Three Ways to Load Specs

1 Recommended

By Chat

User: "Create a feature for user authentication
       with OAuth2, MFA, and password reset"

SDD:  Creates specs/001-user-auth/spec.md with:
      - FR-001: Email/password registration
      - FR-002: OAuth2 Google sign-in
      - FR-003: MFA via TOTP
      - FR-004: Password reset via email
      - US-001 to US-003
      - SC-001 to SC-003

Just describe what you need. SDD generates formal specs with FR/US/SC identifiers. Best for new features and rapid ideation.

2 Batch

Dated Folder

1. Create: workspace/2026-03-23-api-specs/
2. Drop your files (PDF, YAML, HTML, images)
3. Run: /sdd:capture --scan
4. SDD creates rag-memory-of-*.md per file
5. Then: /sdd:spec (uses RAG memory as context)

Best when you have existing requirements documents. The dated folder (YYYY-MM-DD-slug) keeps inputs organized. The workspace/ directory is gitignored.

3 Direct

File Capture

/sdd:capture /path/to/requirements.pdf
/sdd:capture /path/to/api-spec.yaml
/sdd:capture https://docs.example.com/api

Point SDD at any file or URL. It auto-detects the type (text, HTML, image, audio, PDF, slides), then creates a RAG memory with abstract, key takeaways, and full content.

Context for /sdd:spec, /sdd:plan, etc.

Text / Code

Verbatim copy in fenced code block, up to 1000 lines. Language auto-detected.

HTML

Structure summary: title, CSS tokens, headings hierarchy, scripts. Truncated source appended.

Image

Description placeholder generated by LLM when /sdd:capture executes. Dimensions and format noted.

Audio

Transcription placeholder. Duration and codec noted. Full text generated at capture time.

PDF

Page count, section titles extracted, table of contents reconstructed. Content indexed per-section.

Slides

Slide-by-slide recap placeholder. Title, bullet points, and speaker notes per slide.

Ambient Intelligence

Heartbeat in Action

The heartbeat runs in under 100ms on every prompt. It stays silent when healthy (30-minute suppression after a clean run). It only speaks when it finds issues.

claude — my-project
User: "Show me the pipeline status"

⚡ SDD: 3 stale, 1 missing — /sdd:sentinel

This ambient alert was injected by the heartbeat (UserPromptSubmit hook).
It detected:
  - 3 artifacts older than 7 days (specs/002-notifications/)
  - PREMISE.md is missing

Run /sdd:sentinel for full details, or /sdd:insights for recommendations.

Under 100ms

Pure shell script, zero LLM calls. Checks file timestamps and structure only.

Silent When Healthy

30-minute suppression after a clean sentinel run. Never interrupts your flow.

Per-Prompt Hook

UserPromptSubmit fires on every input. Context injected before the LLM sees your message.

Version Control

GitHub Sync

SDD detects your Git remote during initialization and monitors sync status on every sentinel run.

Synced
> /sdd:init

Step 6: Git repo with remote: github.com/user/my-project
  ✓ Sync OK — local matches remote

Step 7: .gitignore created with SDD defaults
Diverged
> /sdd:init

Step 6: Git repo with remote: github.com/user/my-project
  ⚠ Diverged: 2 ahead, 3 behind remote
