¿Por qué SDD?

Método completo de especificación primero con puertas de calidad mandatorias, inteligencia ambiental y trazabilidad bidireccional.

🛡

Puertas Mandatorias G1/G2/G3

Tres puertas de calidad que detienen el pipeline ante violaciones. No son advertencias — son paradas reales.

🔒

Hashing Criptográfico

SHA-256 sobre escenarios BDD. Se genera en Fase 4, se verifica en Fase 7. Detecta manipulación de aserciones.

🏗

Pipeline de 9 Fases

De Constitución a GitHub Issues. Cada fase actualiza el estado del pipeline para prevenir saltos.

💛

Heartbeat Ambiental

Inteligencia por prompt. < 100ms. Escaneo limitado a 50 archivos. Silencioso cuando todo está bien.

🕸

Grafo Bidireccional

Principios → Requisitos → Pruebas → Tareas. Detecta huérfanos en ambas direcciones y referencias rotas.

📋

Workspace por Tarea

Carpetas fechadas con inputs, RAG, logs y tasklog. Escrituras atómicas y concurrencia por flock.

✅

Validación de Esquemas

JSON schemas para context, session y gates. Validadores de contenido para spec, plan y tasks.

🎮

ALM Command Center

Dashboard multi-página: salud, pipeline, specs, calidad, inteligencia, workspace, gobernanza, logs.

🧠

Memoria RAG Segura

Límite 10 MB, resolución de symlinks, detección de binarios. Enrutamiento automático al workspace.

El Pipeline

9 fases, 3 puertas mandatorias. Cada artefacto es trazable y validado.
