Resumen ejecutivo · TL;DR

Del visitante anónimo al badge verificable

El campus une seis flujos en un solo recorrido sistémico. Cada transición cruza una frontera de contexto (identity → catalog → delivery → enrollment → learning → credentials) y emite eventos xAPI que alimentan analítica pedagógica y at-risk predictivo.

Los 6 flujos anclan los 16 entregables del discovery: si un flujo no cierra métrica 100 Check, el roadmap lo absorbe como hipótesis HDD testable.

DDL de enrollment.enrollment (fragmento)

PostgreSQL · schema enrollment
CREATE TABLE enrollment.enrollment (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    person_id uuid NOT NULL REFERENCES identity.person(id),
    course_run_id uuid NOT NULL REFERENCES delivery.course_run(id),
    status text NOT NULL CHECK (status IN ('pending','active','completed','refunded')),
    paid_at timestamptz,
    created_at timestamptz DEFAULT now(),
    UNIQUE (person_id, course_run_id)
);
Regla RLS clave

enrollment es visible solo cuando person_id = auth.uid() o el usuario tiene role_assignment.role IN ('admin','instructor'). Idempotencia del webhook garantizada con ON CONFLICT (stripe_event_id) DO NOTHING.

Login → Dashboard → Lección → Attempt → xAPI → Progreso en vivo

DUA/UDL 3.0 codificado en el modelo de datos

Representation variants

{video, audio, transcript, sign_language?, braille?}. Al menos 2 obligatorias por block.

Expression variants

{text, voice, drawing, code}. El aprendiz elige cómo demuestra lo aprendido.

Engagement hooks

{prompt_reflection, peer_comment, gamification_xp}. Opt-in por cohorte.

Vista learning.v_student_risk — heurística M1

PostgreSQL · heurística inactividad + failed_attempts + % completado
CREATE VIEW learning.v_student_risk AS
SELECT
  e.id AS enrollment_id,
  e.person_id,
  e.course_run_id,
  COALESCE(EXTRACT(DAY FROM now() - MAX(xs.occurred_at)), 999) AS days_inactive,
  COUNT(*) FILTER (WHERE xs.verb = 'failed') AS failed_attempts,
  ROUND(100.0 * SUM(CASE WHEN xs.verb='completed' THEN 1 ELSE 0 END) /
        GREATEST(cr.total_blocks, 1), 1) AS pct_completed,
  CASE
    WHEN EXTRACT(DAY FROM now() - MAX(xs.occurred_at)) > 10 THEN 'at_risk'
    WHEN COUNT(*) FILTER (WHERE xs.verb='failed') >= 3 THEN 'struggling'
    ELSE 'on_track'
  END AS risk_level
FROM enrollment.enrollment e
JOIN delivery.course_run cr ON cr.id = e.course_run_id
LEFT JOIN learning.xapi_statement xs ON xs.person_id = e.person_id
WHERE e.status = 'active'
GROUP BY e.id, e.person_id, e.course_run_id, cr.total_blocks;
